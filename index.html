Am I “Cloud”

Sep 24th, 2013

I read an article, that is about the demanding for Cloud Engineer has significantly increased over the past few years.

The first thing that came to my mind, what is Cloud Engineer?

After google around, it seems the first thing that you need to know what is SaaS, Software as a Service.

With the emerging and maturing of CSS3, HTML5 and JavaScript, there has been a lot of SaaS providers running now in the market, popular providers such as GitHub, Google Doc, Google Drive, DropBox and etc.

Now, I am asking myself, am I Cloud?

For one second, I wish to be the Cloud Strife in Final Fantasy 7, cool, smart, strong and brave.

I have been using DropBox and Box as my storage media.

I have using GitHub as my blog, can I call myself Cloud Blogger?

I am using Google Doc for my office work.

Well, I guess I am only considered as Cloud …. user and not a Cloud Engineer, hey, wait a second…., I am developing my game on AWS.

I would definitely dig more into Cloud Engineer as I want to be Cloud.

Bitbucket vs. GitHub

Sep 23rd, 2013

Just came across this article at the end of the work.

In summary that caught my eyes in the article,

We ended up using both GitHub and Bitbucket to get the best of both worlds. For our private repositories, we use Bitbucket because the pricing model is so much smarter for us. Speed Up With Base64 Images

Sep 23rd, 2013

A very challenging task in web-based game is to get the images download to the player browser client.

As we all know that, using either CSS background-image with URL, JavaScript Image object or HTML <img> tag requires the browser client to initiate new HTTP call for every image download.

Why is this a bad idea?

Every new initiated HTTP call download perform the standard TCP connection and that is heavy, especially in mobile environement as it utilized more processing power.

Besides that, it is much much slower that you can imagine.

Lastly, the request for image download does choke up your websever.

Following is an image showing the number of connections and time spent.

Base64 Image
